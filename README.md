# "复活"物品流通平台

## 开发背景

### 项目简介
"复活"物品流通平台是一个基于Python和Tkinter开发的校园闲置物品交易系统。该平台旨在促进物品的循环利用，让闲置物品重新焕发价值。

我们的口号是：**让生活重启，让闲置复活**。

### 解决的问题
- **资源浪费**：校园中大量闲置物品被丢弃或闲置
- **信息不对称**：学生之间缺乏有效的物品交换渠道
- **环境友好**：促进资源循环利用，减少浪费
- **经济实惠**：为学生提供低成本获取所需物品的途径

### 设计理念
- **用户友好**：简洁直观的操作界面
- **模块化设计**：便于功能扩展和维护
- **数据安全**：完善的用户认证和权限管理
- **教育价值**：培养学生的环保意识和资源共享习惯

## 用户使用指南

### 环境要求
- Python 3.6+
- 标准库依赖：tkinter, json, os, uuid, datetime（均为python内置标准库，用户一般不需安装其他第三方库）

### 安装与运行
1. **下载项目文件**：在完成环境配置后，用户可以通过以下两种方式运行软件：
   
   -**运行封装后的程序**：用户可以下载dist文件夹，解压缩后直接运行“ResurrectionApp.exe”程序，即可进入“复活”软件登录界面。
   
   -**运行源代码**：用户或开发者可以下载源代码后（确保controllers.py、entities.py、main.py、gui_ui.py四个文件在同一目录下），在终端中输入：
   ```
   python main.py
   ```
   即可进入“复活”软件登录界面。

2. **如何成为平台的一员？**

   -**注册**：初次使用需要注册新账号。启动程序后进入登录界面，点击"注册"按钮创建新账号，填写用户名、密码和联系方式，即可完成注册。

   -**登录**：注册成功后，需要在登录界面登录。使用结束后，可以点击“退出登录”登出当前账号，再点击“退出”可以退出软件。

   -**个人信息管理**：若用户希望修改个人信息，可在“个人信息管理”中进行修改。

4. **怎样在平台上发布并管理我的闲置物品？**
   
   -**发布物品**：普通用户成功登陆后，进入主界面，可以点击“发布物品”，填写物品的名称、类型、属性、描述与联系方式等信息，发布自己想要处理的闲置物品。
   
   -**管理物品**：完成发布后，用户可以在“我的发布”中查看自己发布的闲置物品，选中物品后，进行修改、删除等操作。

   -**闲置集市**：用户可以在“闲置集市”中查看所有平台用户发布的闲置物品，也可以选择特定的类型，输入关键字检索自己感兴趣的闲置物品。对于感兴趣的闲置物品，用户可以点击相应条目，在新窗口中浏览物品的详细信息。

5. **除了闲置，“复活”平台还能做什么？**

   -**发布需求**：除了发布闲置物品外，用户还可以发布自己的需求。用户在主界面点击“发布需求”，填写需求类型、描述和联系方式后，即可在平台上发布自己的需求。
   
   -**赏金客栈**：用户可在“赏金客栈”中看到其他用户的需求，双击需求可查看详情。

6. **平台生态建设**
   
    -**申请修改物品类型**：我们欢迎用户对平台现有的物品类型提出改进意见，用户可在主界面点击“申请修改物品类型”，填写对平台的建议，后台收集后会反馈给管理员，管理员审批后会择优采纳。

   -**申请成为管理员**：我们欢迎普通用户申请成为管理员，加入平台建设中。用户可在主界面点击“申请成为管理员”，填写理由后提交申请，后台将由管理员审批后对相应用户开放管理员权限。

   -**成为管理员，我的任务有什么？** 管理员的任务主要为处理其他用户的申请，包括成为管理员的申请和修改物品类型的申请。管理员可在“管理员功能”中进行操作。

## 开发者指南

### 目前版本概述

目前版本为试用版，用户、需求与物品数据暂时只能存储在本地，后续会考虑借入数据库等基础设施。

开发者可以在docs文件夹中查看软件需求分析文档。文档内容包括文档概述、软件的一般性描述、软件的功能需求等。在“软件的功能需求”版块，开发者可以查看采用UML语言绘制的用例图、顺序图以及类图。

### 技术架构

-**编程语言**：python 3.x

-**数据存储**：JSON文件存储于本地

-**GUI框架**：GUI框架：Tkinter

### 核心模块说明
```
controllers.py    # 业务逻辑控制器
├── AuthController    #用户认证和权限管理
├── ItemController    #物品的增删改查等操作
├── TypeController    #物品类型管理
├── DemandController  #需求管理
└── ApplicationController #申请处理

entities.py       # 数据实体定义
├── User         #用户实体
├── Item         #物品实体
├── ItemType     #物品类型实体
├── Demand       #需求实体
└── Application  #申请实体

gui_ui.py         # 用户界面实现
├── LoginUI      #登录界面
├── MainUI       #主界面
├── 各种功能对话框 #包括物品的增删改查界面、需求的发布与查看界面、类型管理界面等
└── 主题和字体管理 #当前版本设定为基于莫兰迪色系的简约主题风格

main.py           # 应用程序入口和主循环
```
